传统设计流程已死——一位 Anthropic IC 的观察

整理自 Jenny（Anthropic 设计 IC）的分享，关于 AI 如何改变设计实践。

传统设计流程已死

“设计师一直被教的那套设计流程，我们几乎把它当成’教条’在信。现在它基本已经死了。”

“今天做设计，你根本没有时间再去做那些精美、打磨到极致的高保真稿了。工程师可以同时拉起七个 Claude 快速做功能，作为设计师，你必须放下那套流程。”

“其实在 AI 之前，它就已经在走下坡路了。”

快速发布 vs 深度调研

“能快速执行、把东西做出来、用真实数据去试、以真实用户为中心去验证，这确实会带来更好的产品。”

“我们现在做的很多东西，是基于非确定性的 AI 模型。你没法把所有状态都 mock 出来，也没法光靠理论推演。你必须接上真实模型，用真实用户的用法去跑，才能知道会发生什么。”

“AI 的用例，很多时候不是你事先’设计出来’的，是你看着用户怎么用、边看边发现的。所以执行与迭代非常关键。”

时间分配的变化

几年前：

  • 60%–70% 的时间做 mock 和原型
  • 20% 左右跟工程师一起 jam、咨询式配合
  • 10% 协调类会议

现在：

  • 30%–40% 做 mock
  • 30%–40% 直接跟工程师配对一起做
  • 剩下的是实现——真的去做、去交付、去发布

AI 工具栈

“我们基本就是在 Claude 全家桶里深挖。”

“我几乎把所有’聊天类需求’都迁到 Cowork 了，因为它更适合长任务、长流程的事情。Claude Code 我主要是在 VS Code 里用，因为我经常要改前端细节，能看着代码再跟 Claude 对话会很方便。”

“有人说’这个 icon 偏了’，你直接 @mention Claude，Claude 就给你改了，然后你把 PR 捡起来一看：搞定。”

三类最有竞争力的设计师

强通才

“不是那种’很多都懂一点’的通才，而是更像’块状能力’。在几个核心技能上都能做到接近行业 80 分位的水平，很扎实、很能打。设计角色正在被拉伸，越来越像 PM，也越来越像工程。”

深度专家

“非常技术向，甚至能做到 50% 像软件工程师。视觉设计特别强的人，比如图标。在一个’人人都能做出东西’的世界里，这种很深的、强工艺的能力，反而能让你做出来的东西真正有辨识度。”

Craft New Grad

“职业早期的人，但给人的感觉’聪明得超出年龄’。学得快、愿意学新打法、不带太多固定仪式感和流程包袱。我们这些在行业里待久的人，身上会背着很多预设和期待，而他们没有。”
